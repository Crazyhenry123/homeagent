# HomeAgent — API Reference

**Base URL:** `http://<ALB-DNS-NAME>`

All endpoints are prefixed with `/api/` except the health check. Admin endpoints are under `/api/admin/`.

## Authentication

Most endpoints require a Bearer token obtained during registration:

```
Authorization: Bearer <device_token>
```

The token is a 64-character base64url string generated by `secrets.token_urlsafe(48)`.

### Roles

- **member** — Standard user. Can chat, manage own profile, toggle available agents.
- **admin** — Full access. Can manage members, agents, templates, health records, family tree.

## Auth

### Register Device

```
POST /api/auth/register
```

No authentication required. Requires a valid, unused invite code.

**Request:**
```json
{
  "invite_code": "AB12CD",
  "device_name": "iPhone 15 Pro",
  "platform": "ios",
  "display_name": "Mom"
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `invite_code` | string | Yes | 6-char hex code, case-insensitive |
| `device_name` | string | Yes | Human-readable device identifier |
| `platform` | string | Yes | `"ios"`, `"android"`, or `"web"` |
| `display_name` | string | Yes | User's display name |

**Response (201):**
```json
{
  "user_id": "01JXYZ...",
  "device_token": "abc123...xyz789"
}
```

### Verify Token

```
POST /api/auth/verify
```

**Auth:** Bearer token required.

**Response (200):**
```json
{
  "valid": true,
  "user_id": "01JXYZ...",
  "name": "Mom",
  "role": "admin"
}
```

### Create Invite Code (Admin)

```
POST /api/admin/invite-codes
```

**Auth:** Admin required.

**Response (201):**
```json
{
  "code": "AB12CD",
  "expires_at": "2099-12-31T00:00:00+00:00"
}
```

## Chat

### Send Chat Message (SSE)

```
POST /api/chat
```

**Auth:** Bearer token required.

**Request:**
```json
{
  "message": "How do I fix a leaky faucet?",
  "conversation_id": "01JXYZ...",
  "model": "us.anthropic.claude-opus-4-6-v1"
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `message` | string | Yes | User's message text |
| `conversation_id` | string | No | Omit to start a new conversation |
| `model` | string | No | Override default Bedrock model ID |

**Response (200):** `Content-Type: text/event-stream`

```
data: {"type":"text_delta","content":"You can ","conversation_id":"01JXYZ..."}
data: {"type":"text_delta","content":"fix a leaky faucet by ","conversation_id":"01JXYZ..."}
data: {"type":"message_done","conversation_id":"01JXYZ...","message_id":"01JABC..."}
```

| Event Type | Fields | Description |
|------------|--------|-------------|
| `text_delta` | `content`, `conversation_id` | Chunk of the assistant's response |
| `message_done` | `conversation_id`, `message_id` | Stream complete, message saved |
| `error` | `content` | Error during streaming |

## Conversations

### List Conversations

```
GET /api/conversations?limit=20&cursor=<updated_at>
```

**Auth:** Bearer token required. Returns only the authenticated user's conversations.

**Response (200):**
```json
{
  "conversations": [
    {
      "conversation_id": "01JXYZ...",
      "user_id": "01JABC...",
      "title": "How do I fix a leaky faucet?",
      "created_at": "2026-02-28T12:00:00+00:00",
      "updated_at": "2026-02-28T12:05:00+00:00"
    }
  ],
  "next_cursor": "2026-02-28T11:50:00+00:00"
}
```

### Get Conversation Messages

```
GET /api/conversations/<conversation_id>/messages?limit=50&cursor=<sort_key>
```

**Auth:** Bearer token required. Must own the conversation.

**Response (200):**
```json
{
  "messages": [
    {
      "conversation_id": "01JXYZ...",
      "sort_key": "2026-02-28T12:00:00+00:00#01JABC...",
      "message_id": "01JABC...",
      "role": "user",
      "content": "How do I fix a leaky faucet?",
      "created_at": "2026-02-28T12:00:00+00:00"
    }
  ],
  "next_cursor": "..."
}
```

### Delete Conversation

```
DELETE /api/conversations/<conversation_id>
```

**Auth:** Bearer token required. Must own the conversation.

**Response (200):**
```json
{"success": true}
```

## Agent Templates (Admin)

### List Templates

```
GET /api/admin/agent-templates
```

**Auth:** Admin required.

**Response (200):**
```json
{
  "templates": [
    {
      "template_id": "01JXYZ...",
      "agent_type": "health_advisor",
      "name": "Health Advisor",
      "description": "Comprehensive family health companion...",
      "system_prompt": "You are a...",
      "default_config": {"safety_disclaimers": true},
      "is_builtin": true,
      "available_to": "all",
      "created_by": "system",
      "created_at": "...",
      "updated_at": "..."
    }
  ]
}
```

### Create Template

```
POST /api/admin/agent-templates
```

**Auth:** Admin required.

**Request:**
```json
{
  "name": "Meal Planner",
  "agent_type": "meal_planner",
  "description": "Plans weekly meals for the family",
  "system_prompt": "You are a meal planning assistant...",
  "default_config": {},
  "available_to": "all"
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `name` | string | Yes | Display name |
| `agent_type` | string | Yes | Unique slug (e.g. `meal_planner`) |
| `description` | string | Yes | What the agent does |
| `system_prompt` | string | Yes | Full system prompt for the sub-agent |
| `default_config` | object | No | Default config merged into AgentConfigs |
| `available_to` | string\|list | No | `"all"` or list of user_ids. Default: `"all"` |

**Response (201):** Template object.

### Update Template

```
PUT /api/admin/agent-templates/<template_id>
```

**Auth:** Admin required. Can update `name`, `description`, `system_prompt`, `default_config`, `available_to`.

### Delete Template

```
DELETE /api/admin/agent-templates/<template_id>
```

**Auth:** Admin required. Rejects built-in templates. Cascade-deletes all AgentConfigs referencing this agent_type.

## Member Agent Self-Service

### List Available Agents

```
GET /api/agents/available
```

**Auth:** Bearer token required. Returns templates available to the calling user, with `enabled` flag.

**Response (200):**
```json
{
  "agents": [
    {
      "template_id": "01JXYZ...",
      "agent_type": "health_advisor",
      "name": "Health Advisor",
      "description": "...",
      "is_builtin": true,
      "enabled": false
    }
  ]
}
```

### List My Agent Configs

```
GET /api/agents/my
```

**Auth:** Bearer token required.

**Response (200):**
```json
{"agent_configs": [...]}
```

### Enable Agent

```
PUT /api/agents/my/<agent_type>
```

**Auth:** Bearer token required. Validates agent is available to the user.

**Errors:** 403 if agent is not available to the user.

### Disable Agent

```
DELETE /api/agents/my/<agent_type>
```

**Auth:** Bearer token required.

## Family Tree (Admin)

### List All Relationships

```
GET /api/admin/family/relationships
```

### Get User's Relationships

```
GET /api/admin/family/relationships/<user_id>
```

### Create Relationship

```
POST /api/admin/family/relationships
```

**Request:**
```json
{
  "user_id": "01JABC...",
  "related_user_id": "01JDEF...",
  "relationship_type": "parent_of"
}
```

Valid types: `parent_of`, `child_of`, `spouse_of`, `sibling_of`. Bidirectional relationships are auto-created.

### Delete Relationship

```
DELETE /api/admin/family/relationships/<user_id>/<related_user_id>
```

## Health Records (Admin)

### List Records

```
GET /api/admin/health-records/<user_id>?record_type=<type>
```

### Create Record

```
POST /api/admin/health-records/<user_id>
```

**Request:**
```json
{
  "record_type": "condition",
  "data": {"name": "Asthma", "severity": "mild"}
}
```

Valid record types: `condition`, `medication`, `allergy`, `procedure`, `immunization`, `vital_sign`, `lab_result`.

### Get/Update/Delete Record

```
GET    /api/admin/health-records/<user_id>/<record_id>
```

### Audit Log

## Health Documents (Admin)

### List Documents

```
GET /api/admin/health-documents/<user_id>
```

### Upload Document

```
POST /api/admin/health-documents/<user_id>/upload
```

**Request:**
```json
{
  "filename": "lab-results.pdf",
  "content_type": "application/pdf",
  "file_size": 102400,
  "record_id": "01JXYZ...",
  "description": "Annual blood work"
```

Returns a pre-signed S3 upload URL.

```
GET /api/admin/health-documents/<user_id>/<document_id>/download
```

Returns a pre-signed S3 download URL.

### Delete Document

```
DELETE /api/admin/health-documents/<user_id>/<document_id>
```

## Error Response Format

All error responses follow this structure:

```json
{
  "error": "Human-readable error message"
}
```

## Rate Limits

No application-level rate limits are currently enforced. The system relies on:
- AWS ALB connection limits
- DynamoDB on-demand throughput
- Bedrock model invocation quotas
